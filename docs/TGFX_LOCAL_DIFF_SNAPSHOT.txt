TGFX local checkout diff snapshot
================================

Purpose
-------
This file preserves the real source deltas found in the local TGFX checkout at:

  /Users/mshonle/Projects/Tencent/tgfx

It was created before resetting that checkout back to HEAD so the edits would
not be lost. The local Git worktree was in a broken metadata state when this
was captured: `.git/index` was missing, so `git status` misleadingly showed the
entire repository as deleted and untracked at the same time. To isolate the
actual edits, the working tree was compared against `git archive HEAD`.

Observed repository state at capture time
-----------------------------------------
- Branch: `main`
- HEAD: `6ea71127be6649a711a0271e0fda16f1de17ab74`
- Origin: `https://github.com/Tencent/tgfx.git`
- `.git/index` missing
- `git fsck --no-dangling --connectivity-only` passed
- No Git submodules were present (`.gitmodules` absent)
- TGFX dependency bootstrap uses `sync_deps.sh` / `depsync`, not submodules

Interpretation
--------------
These edits look like an attempt to reduce or work around macOS/OpenGL-related
warnings and API issues:

- `CMakeLists.txt` relaxes Clang's `-Werror` handling.
- `DataView.h` adds `<stddef.h>`.
- `CGLDevice.*` removes `NSOpenGLContext` ownership in favor of `CGLContextObj`.

The risky change is `src/gpu/opengl/cgl/CGLWindow.mm`, where the local version
stops creating an OpenGL surface and makes present a no-op. That may be
acceptable only if a different rendering/backend path is used in practice, such
as Metal, but it is not behaviorally equivalent to HEAD for the OpenGL CGL
window path.
